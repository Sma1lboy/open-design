§ 02 — the stack

Three layers.
One quiet night.

Every role a human team would fill — PM, TL, dev, reviewer — compressed into three agent layers that talk in structured JSON and never need lunch.

sprint planning is deterministic
worker output goes through a second-pass reviewer
every diff is a real PR on a real branch
stuck tasks escalate, not retry forever
01 · CONDUCTOR
The one in charge.

Reads your backlog, decides order, calls the shots. Reviews every PR before it merges. Escalates to you on the morning digest if a task is genuinely stuck.

02 · SPRINT MASTER
The middle manager.

Breaks each sprint into tasks a single worker can finish in one context window. Avoids the "200-file PR" failure mode. Routes by file ownership heuristic.

03 · WORKER
The one writing code.

Opens a branch, writes the patch, runs tests locally, opens a PR. Speaks only in diffs. Doesn't talk to other workers. If blocked, files a ticket and moves on.

Is it safe to run unattended? ›

Every change lands on a branch, never on main. You review in the morning. There is no auto-deploy. It is a PR factory, not a push-to-prod bot.

Does it cost what OpenAI charges? ›

Yes. You bring your own keys. Typical night: $2–$8 depending on sprint size. Set a --budget cap and it stops.
