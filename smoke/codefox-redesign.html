to what runs.

Say what you want in plain English. No schema language. No setup wizard. "A booking tool for my yoga studio" is enough.

The Planner drafts pages. The Schema Agent proposes tables. The Stylist picks tokens. The Reviewer argues. You watch.

One opinion each.

Instead of one model doing everything badly, we run four specialists who argue until the plan is real. You can read the transcript.

01
The Planner

§ chapter five

Questions we
keep getting.

Honest ones. If yours isn't here, the discord is small enough that Jackson will probably answer.

Does it actually work or is this a demo? +

It works. 428 apps were built today. The median one deployed. Here's the receipt.

Do I get the source code or is it a black box? +

You own the repo from the moment it builds. Self-host reads directly to disk. Studio exports to GitHub in one click.

What stack does it generate? +

Next.js 14 (App Router), Drizzle + Postgres, Tailwind, shadcn components, auth via Lucia. Swappable — the Planner will argue with you if you pick something it thinks won't work.

How is it different from v0, Bolt, Lovable? +

Those are (great) landing-page builders. Codefox generates a full-stack app — auth, DB, API, UI — as one repo, with four argued agents instead of one opinionated model. Plus: it's open source. You're never locked in.

I don't like the current codefox design. Is this the new one? +

Yes. This is the one. We burned the purple gradients and started with a warm cream + rust palette, a variable grotesque in condensed mode, and type that looks like it came out of an editorial studio, not a template. If this isn't right either, the Stylist agent will happily try again — we put it in the app on purpose.
