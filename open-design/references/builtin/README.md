# Builtin — upstream builtin skills

Four upstream builtin skills brought over verbatim. Each is a self-contained craft rule-set for a specific artifact family. Read the matching one in full before generating that artifact type.

| File | When to read |
|---|---|
| `frontend-design-anti-slop.md` | Any visual artifact. Overlaps with `../anti-slop.md` but adds specific pairing suggestions + tone-commitment rule. Read once per session. |
| `pitch-deck.md` | Slide decks, pitch presentations, keynotes. |
| `data-viz-recharts.md` | Dashboards where you choose Recharts over pure SVG. |
| `mobile-mock.md` | Mobile app screens, iOS / Android mockups. Pairs with `../ios-starter.md`. |

These are intentional duplication with the core references — the builtin skills are more opinionated and domain-specific, while `references/` proper holds the general methodology.

Precedence if they conflict: the builtin skill wins for its narrow domain; core references win for anti-slop / structure questions.
